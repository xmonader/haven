Ref-scoped recipients: a secret encrypts to the active readers of its ref. **Server-side enforcement (M7c):** signed-request auth (X-Haven-Pub/Time/Sig, 300s skew) maps key→actor; gates `GET /refs` (filter readable), `GET /objects` (reachability from readable refs), `POST /refs` (write grant; `refs/policy` requires valid chain extension — no history rewrite), `PUT /objects` (members only). Tests + vet green.
  **Hardening pass** — body- and nonce-bound request signatures (sign method+path+time+sha256(body)+nonce; closes ref-tamper and verbatim-replay, even within the skew window), per-actor reachable-object cache on the server (O(repo) once, not per object), drift baseline armed at first commit, TLS serving (`--tls-cert/--tls-key`), symlink tracking (mode 120000), rename-aware three-way merge, porcelain `reset`/`restore`/`tag`, and unit tests for remote/index/config/lock/identity. 13 test packages green, vet clean.
  **Porcelain pass** — `cherry-pick`/`revert` (shared `threeWayApply` onto HEAD), linear `rebase` (replay base..HEAD onto upstream via `applyOnto`; all-or-nothing roll-back on conflict), `stash` save/list/pop (off-branch commit under `refs/stash`, three-way restore), manual `bisect` (start/good/bad/reset over a `refs/bisect/work` ref). 13 test packages green, vet clean.
  **Production hardening pass** — fixed: `secret rotate` was a storage no-op (PutRaw INSERT OR IGNORE; now ReplaceContent) and didn't propagate over push (server now upserts secret content via PutSecret); ref updates were a check-then-set TOCTOU (now atomic `ref.CompareAndSwap`, concurrency-tested); replay nonces moved from in-memory to a durable `seen_nonces` table; server gained request logging + graceful shutdown; fuzz tests on the commit/tree parsers (~260k execs, no panics). Crash-consistency: every op writes objects before advancing the ref (the single atomic commit point), so interruption leaves only gc-able orphans.
  **Production-grade pass** — operational scaffolding + an adversarial review of the secret/crypto data path. Added: `hv version` (link-time injected), a hardened Makefile (`race`/`cover`/`dist` release targets), GitHub Actions CI (build/vet/gofmt/`-race`/coverage/parser-fuzz on Linux+macOS) and a tag-driven release workflow, `LICENSE` (MIT), `SECURITY.md` (private disclosure), `CONTRIBUTING.md`, and `docs/threat-model.md`. Review-driven fixes, each with a regression test: decrypted secrets now 0600 (were world-readable 0644); decrypted plaintext verified against the object hash before writing (forged-ciphertext substitution refused); decryption errors no longer masked as "not a recipient" (no silent data loss); downloaded non-secret objects verified against the requested hash; tree walks bounded (`maxTreeDepth`) against stack-overflow DoS; nonce eviction keyed on server receive time; private-key file/dir perms enforced; `serve --policy-root` pins the bootstrap root against open-server takeover; clone aborts on a corrupt policy object. `internal/workspace` went from 0% to covered on the write path. All 17 test packages green under `-race`, gofmt- and vet-clean.
  **Hardening follow-ups (closed in code):** request signatures now bind the target host and `serve --origin` enforces it, closing cross-server replay under a shared keyring; the working-copy lock is cross-platform (flock on Unix, `LockFileEx` on Windows) so Haven builds and runs on Windows too — release binaries now cover linux/darwin/windows × amd64/arm64.
  **Delta storage pass** — implemented delta/packfile compression (the last code-actionable scale item). `internal/store/delta.go`: copy/insert `MakeDelta`/`ApplyDelta` (validated, rejects malformed/truncated/circular). `codec.go`: `codecDelta` envelope (`EncodeDelta`/`DecodeDelta`). `object.Store`: delta-aware `Get`/`Each` (depth-bounded by `maxDeltaDepth`), `StoreAsDelta` (self-verifies byte-for-byte before rewrite, no-op when it wouldn't shrink, refuses delta/self bases → depth-1 chains), `DeltaBase`/`StoredSize`/`Metas`. New `hv repack` (`cmd_repack.go`): groups whole blobs/trees/commits by type+size, deltas against a recent same-type base, then `VACUUM`s; secrets and the policy chain left whole. `gc` retains the base of every reachable delta (`addDeltaBases`). The wire is untouched — push/fetch and the server's `getObject` read via `store.Get`, so only whole objects ever cross the wire. Measured: 5-revision fixture `.haven/haven.db` 98304→86016 bytes, repack idempotent, fsck clean after repack+gc. Tests: `store/delta_test.go`, `object` delta tests, `cli/repack_test.go` (e2e integrity + gc-keeps-base). All 17 packages green under `-race`, gofmt/vet clean.
  The only remaining pre-"production-grade-audited" item is an **external security audit** (cannot be self-performed — a third party must do it; `docs/audit-readiness.md` is prepared to enable it). Streaming reads for huge single blobs (§16 step 3) remain future scale work, not a defect.
  **Production blockers pass** — closed four verified holes: (1) unbounded `io.ReadAll(r.Body)` was a memory-exhaustion DoS → `http.MaxBytesReader` cap (`MaxRequestBytes` 256 MiB, injectable for tests); (2) no schema migration despite the claim → `PRAGMA user_version` versioning with an ordered migration runner that refuses a DB newer than the binary; (3) objects stored uncompressed → at-rest zlib codec (1-byte tag, raw fallback so incompressible data never grows) with a v1→v2 migration that rewrites legacy rows; (4) any member could overwrite any secret's ciphertext via the PutSecret upsert → a content-changing rewrite now requires write access to a ref reaching the secret (identical bytes stay idempotent). All 16 test packages green under `-race`, vet clean.
- **Known limitations (by choice / not yet):** objects are zlib-compressed and `hv repack` delta-compresses similar objects, but each object is still loaded whole into memory (streaming reads are future — fine for source, not huge single binaries); shared-plaintext secrets across refs store one ciphertext (benign — identical plaintext, self-heals via rotate; NOT ref-scoped because that would cause spurious merge conflicts on shared `.env`); octopus merge (>2 parents) not implemented.
- **Next:** streaming reads for huge blobs, per-process→shared reachability cache, metrics. None block solo/small-team use. External security audit is the only gate to "production-grade audited".
- **Blocked:** nothing.

## Architecture Notes
- `internal/store`: owns the DB connection + schema (WAL, foreign keys, busy_timeout).
- `internal/repo`: `.haven/` layout, Init/Open/discover (walks up for `.haven`), HEAD read/write.
- `internal/cli`: `Command` registry + `Dispatch`; one `cmd_*.go` per subcommand.
